# Zoo Management System
## Описание
Данный проект представляет собой информационную систему для учета животных в зоопарке. Основная цель работы — реализация объектно-ориентированной модели данных с четким разделением ответственности между бизнес-логикой, интерфейсом и хранением данных.
## Ключевые концепции ООП
В работе применены фундаментальные принципы ООП:
* **Иерархия и Наследование:** Создан базовый абстрактный класс `Animal`, от которого наследуются три специализированных класса: `Mammal` (Млекопитающее), `Bird` (Птица) и `Fish` (Рыба).
* **Абстракция:** Базовый класс `Animal` содержит абстрактный метод `getType()`, что обязывает всех наследников идентифицировать свой биологический вид.
* **Полиморфизм:** Использование коллекций типа `List<Animal>` позволяет обрабатывать разнородные объекты (птиц, рыб, млекопитающих) единообразно при поиске, фильтрации и сортировке.
* **Инкапсуляция:** Поля данных (имя, возраст, вес) скрыты модификатором `private`. Доступ к ним и их изменение осуществляется строго через публичные методы (геттеры и сеттеры).
## Функционал
Программа поддерживает полный цикл управления коллекцией:
1. **Добавление:** раздельные сценарии для млекопитающих, птиц и рыб.
2. **Просмотр:** вывод полного списка объектов с их характеристиками.
3. **Поиск:** нахождение животных по точному совпадению имени.
4. **Фильтрация:** по биологическому типу, конкретному возрасту и диапазону веса.
5. **Сортировка:** упорядочивание списка по алфавиту (имя) и по возрастанию возраста.
6. **Статистика:** автоматический подсчет количества особей каждого вида.
7. **Управление данными:** редактирование (вес/возраст) и удаление записей.
8. **Безопасность:** ограничение прав администратора (защита паролем `admin123`).
9. **Хранение:** сохранение и автоматическая загрузка коллекции через сериализацию в файл `zoo_data.dat`.
10. **Логирование:** фиксация действий пользователей и ошибок системы через **Apache Log4j2**.
## Структура проекта
* `Main.java` — Точка входа в программу, инициализация приложения.
* `ZooApp.java` — Контроллер интерфейса: реализация консольного меню, обработка ввода пользователя и проверка прав доступа.
* `ZooManager.java` — Класс-менеджер: содержит коллекцию объектов и реализует алгоритмы сортировки, фильтрации и работы с файловой системой.
* `Animal.java` — Файл, содержащий абстрактную модель и классы-наследники (Mammal, Bird, Fish).
## Требования и запуск
1. Java Development Kit (JDK) 11+.
2. Библиотеки **Log4j 2.25.3**.
3. Размещение файла `log4j2.xml` в директории ресурсов или корне проекта.
### Инкапсуляция
Если проект собран через Maven, выполните:
```text
mvn clean compile
mvn exec:java -Dexec.mainClass="Main"
```
### Данные для доступа:
* **Режим Гостя:** Доступен по умолчанию (только просмотр, поиск, фильтры).
* **Режим Админа:** Пункт меню 13, пароль: `admin123`.
## Обработка исключений
Система обеспечивает стабильность работы через:
* **Валидацию ввода:** Проверка на тип данных (защита от ввода текста вместо чисел).
* **Безопасность индексов:** Проверка существования объекта перед удалением или редактированием.
* **Журналирование:** Все важные события (вход админа, удаление данных, ошибки ввода/вывода) записываются в файл `logs/zoo_app.log` с указанием времени.
